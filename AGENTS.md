## Critical Conventions

### Must Do

- `proxy.ts` — never `middleware.ts` (Next.js 16)
- `@import "tailwindcss"` in CSS (Tailwind v4)
- Object syntax for TanStack Query v5: `{ queryKey: [...], queryFn: ... }`
- Zod v4 top-level types: `z.email()`, `z.uuid()`, `z.url()`
- `{ error: "msg" }` not `{ message: "msg" }` in Zod
- `getClaims()`/`getUser()` for server auth — never `getSession()`
- Business logic in `features/<name>/server/*.ts` — never inline in actions
- Sentry instrumentation for all error paths
- Run `pnpm typecheck && pnpm lint` after changes

### Must NOT

- Pages Router (`pages/`, `getServerSideProps`, `getStaticProps`)
- `middleware.ts` → use `proxy.ts`
- `@supabase/auth-helpers-nextjs`
- `next/router` → use `next/navigation`
- `tailwind.config.*` files
- `useQuery(['key'], fn)` — positional syntax
- `getSession()` for authorization decisions
- Google/Apple OAuth
- `SUPABASE_SECRET_KEY` on client
- Stripe SDK or references
- Class components
- Mock data — use real Supabase
- `console.log` in production code paths

## Native App Forward-Compatibility

MVP uses Server Actions for writes. Before native release, every write SA gets a Route Handler twin reusing the same business helper and Zod schema. Today's rule: keep business logic in `features/<name>/server/*.ts`, Zod schemas in `features/<name>/schemas.ts`, never inline in the SA function body.

## Testing

- Unit/component: Vitest + @testing-library/react
- E2E: Playwright
- HTTP mocks: MSW
- Run: `pnpm test:unit`, `pnpm test:e2e`, `pnpm verify` (all checks)

## Observability

Sentry is MANDATORY for production. All runtime errors, unhandled exceptions, 5xx failures, and Inngest function failures must go to Sentry. PII must be scrubbed. See `docs/14-sentry-observability.md` for full mandate.
